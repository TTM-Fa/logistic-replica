External price API

When you choose External price API as your cost model, Autonomous Quotation does not calculate the predicted buying price itself. Instead, it sends each order’s details to your HTTP service. Your service returns a predicted price; that value is used as the basis for quote calculation (before your rulesets apply margins and other adjustments).

How it works

  • You host an endpoint that accepts POST requests with a JSON body (the order payload).
  • Autonomous Quotation calls that URL when an order needs a price under the external cost model.
  • The request path on your side is expected to match the contract in the API documentation (typically POST /price-prediction on the base URL you configure).

Configuration

  1. Open Configuration → Cost model.
  2. Select External price API as the default model or add an exception that uses it.
  3. Enter your service URL (the full address Autonomous Quotation should call).
  4. Optionally add headers (for example Authorization or API keys). Those headers are sent with every prediction request. Header values are sent exactly as you enter them - for example, include the full Bearer … string in the value if your API expects that format; nothing is added or rewritten for you.

What we send (request body overview)

The JSON body represents the order as needed for pricing. Important groups of fields include:

  • order_id - Internal Autonomous Quotation order identifier.
  • order_source - Where the order comes from (e.g. Autonomous Procurement, Best Carrier, Marketplace).
  • created_at, display_name, optional order_source_reference_id - Identity and display context.
  • customer_data - Customer identifiers and contact details your pricing may depend on.
  • trip_details - Route-related metrics such as driving distance, driving time, and regions crossed.
  • pricing_data - Currency and optional match-now price hints where applicable.
  • laneway - Structured origin and destination (country, postal code, state where available).
  • cargos - Line items with measures (e.g. weight) and required services.
  • activities - Stops, time windows, sites, and services tied to execution.

The exact shape and required fields are defined in the API documentation.

What you must return (success)

Respond with HTTP 200 and a JSON body that includes at least:

  • predicted_buying_price - Number: your predicted cost in the unit implied by currency.
  • currency - String (e.g. ISO currency code), aligned with the order’s pricing context.
  • confidence - Optional number between 0 and 1; if omitted, a default may be assumed per the API spec.

Errors and rejections

If your API responds with any status other than HTTP 200 (or the request fails due to timeout or network errors), Autonomous Quotation treats the prediction as failed and the order is rejected. You can find those orders in the Unprocessed tab. Design your API for a reliable 200 response when you can supply a price.

Authentication

Use the header rows in the cost model configuration to send tokens or API keys your endpoint expects (for example a bearer token or custom header). Values are passed as-is on each request. Only configure secrets in line with your organisation’s security practices.

Testing with real payloads

On Configuration → Cost model, under External price API, use Example payloads next to API documentation to open a list of recent orders. You can search and expand an order to view the JSON body as Autonomous Quotation would send it. That helps you align your implementation with real data.

For full specification, see API documentation.
